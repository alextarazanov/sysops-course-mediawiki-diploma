# Дипломная работа по курсу "Системный администратор"

*Выполнил студент 28 когорты Алексей Таразанов*

## Введение

Цель проекта — подготовить инфраструктуру и успешно запустить сервис, обеспечить его доступность и надёжную работу.

Дополнительная цель проекта — автоматизировать развёртывание с помощью Terraform и Ansible и использовать дополнительные навыки и инструменты вне основного курса.

## Техническое задание

Контекст задачи: руководитель предлагает внедрить документирование с использованием `MediaWiki`. `MediaWiki` — вики-движок, написанный специально для Википедии, но используемый во многих других проектах фонда «Викимедиа», а также частных и государственных организациях. Является свободным ПО и распространяется на условиях GPL. 

* **ОС сервисов проекта**: `Ubuntu 22.04.`
* **Необходимые библиотеки и фреймворки**:
  * `MediaWiki` (основное ПО).
  * `PHP` (требуется для работы `MediaWiki`).
  * Веб-сервер (`Nginx`).
  * `PostgreSQL` (база данных).
* **Планируемая нагрузка (количество клиентов)**:
  * Первоначально — около 40 пользователей (центральный офис).
  * В перспективе — *масштабирование* на всех сотрудников компании, включая офисных и удалённых работников (точное количество уточняется, но *система должна быть готова к росту нагрузки*).
* **Специфика приложения**: Система документооборота на базе `MediaWiki` — вики-платформа для хранения, редактирования и совместной работы с документами. Предназначена для внутреннего использования в компании. Требует поддержки редактирования, версионности, поиска и управления правами доступа.
* **Интерфейс взаимодействия с пользователем (HTTP-сервис)**:
  * Веб-интерфейс, доступный по HTTP.
  * Используется `Nginx` в качестве балансировщика нагрузки и веб-сервера.
  * Планируется работа через браузер (клиентская часть — стандартный веб-интерфейс `MediaWiki`).
* **Вспомогательные сервисы**:
  * В качестве балансировщика будем применять `Nginx`.
  * В качестве базы данных используем `PostgreSQL`.
  * Для резервного копирования БД — `pg_dump`.
  * Выберем систему мониторинга `Zabbix`.

## Архитектура решения

![image](./assets/solution.svg)

*Схема решения*

Где:

* Пользователи — сотрудники компании (сначала 40, потом все), используют браузер для доступа к вики.
* Nginx — выполняет роль обратного прокси и балансировщика нагрузки, распределяя запросы между двумя экземплярами MediaWiki, размещёнными на отдельных виртуальных машинах.
* MediaWiki (2 экземпляра) — веб-приложения на Ubuntu 22.04, используют PHP и Nginx для обработки запросов.
* Кластер PostgreSQL — отказоустойчивый кластер (Master + реплика), хранит все данные вики.
* Резервное копирование — выполняется через `pg_dump` и скрипт, запускаемый по расписанию (`cron`).
* На всех узлах установлен Zabbix Agent, который собирает метрики и отправляет на Zabbix Server.
* На ВМ6:
  * Zabbix Server — централизованно мониторит доступность, нагрузку, ошибки.
  * Хранилище резервных копий.

Вне схемы предполагается ВМ7, локальный компьютер или IaC-репозиторий с настроенными пайплайнами, с которых происходит нарезка виртуальных машин с помощью Terraform и их настройка с помощью Ansible. Часть функциональности, связанной с резервными копиями, реализовал с помощью инструментов Yandex Cloud.

## Пошаговая инструкция по развёртыванию инфраструктуры

### Terraform

Чтобы развернуть инфраструктуру сервиса MediaWiki:

1. Перейдите в домашнюю директорию с помощью команды:

   ```bash
   cd ~/
   ```

1. Клонируйте репозиторий с конфигурацией сервиса с помощью команды:

   ```bash
   git clone https://github.com/alextarazanov/sysops-course-mediawiki-diploma.git
   ```

1. Перейдите в директорию проекта с помощью команды:

   ```bash
   cd wikimedia/
   ```

1. [Установите Yandex Cloud CLI]((https://yandex.cloud/ru/docs/cli/quickstart#install)).

1. Создайте пару SSH-ключей в папке `/ansible/ssh_keys/` и добавьте значение открытого ключа в файл `/terraform/ssh_users` с помощью команд:

   ```bash
   cd ansible/
   mkdir ssh_keys/
   ssh-keygen -f ~/wikimedia/ansible/ssh_keys/ansible_id_ed25519 -t ed25519
   echo "ansible:$(cat ssh_keys/ansible_id_ed25519.pub)" > ../terraform/ssh_users
   ```

1. Перейдите в директорию [`terraform`](./terraform/) и [установите Terraform](https://yandex.cloud/ru/docs/tutorials/infrastructure-management/terraform-quickstart#install-terraform) с помощью команд:

   ```bash
   cd ../terraform/
   sudo apt install unzip
   wget https://hashicorp-releases.yandexcloud.net/terraform/1.13.4/terraform_1.13.4_linux_amd64.zip
   unzip terraform_1.13.4_linux_amd64.zip
   sudo mv terraform /usr/bin/
   ```

1. Скопируйте файл `.terraformrc` в домашнюю директорию с помощью команды:

   ```bash
   cp .terraformrc ~/
   ```

1. Запустите скрипт для авторизации по сервисной учётной записи через `yc` и инициализируйте Terraform с помощью команд:

   ```bash
   ./setup_variables.sh
   terraform init
   ```

1. Запустите развёртывание инфраструктуры в Yandex Cloud с помощью скрипта:

   ```bash
   . deploy_terraform.sh
   ```

### Ansible

Чтобы настроить конфигурацию виртуальных машин с помощью Ansible:

1. Перейдите в директорию [`ansible`](./ansible/) и установите Ansible с помощью команды:

   ```bash
   cd ../ansible/
   sudo apt install ansible
   ```

1. Откройте страницу **Виртуальные машины** в Yandex Compute Cloud.

1. Откройте файл `inventory.yml` и добавьте в IP-адреса созданных виртуальных машин в соответствующие поля `ansible_host` для каждого хоста:

   ```bash
   nano inventory.yml
   ```

   * Для хостов `balancer-1` и `monitoring-1` укажите публичные **IP-адреса**. Для остальных хостов используйте внутренние IP-адреса.
   * IP-адрес хоста `balancer-1` укажите в `balancers:hosts:balancer-1:ansible_host` и в поле `internal:children:wikimedia:databases:vars:ansible_ssh_common_args`.

   После этого список хостов будет готов к применению на них конфигурации Ansible.

1. Запустите плейбук Ansible с помощью команды:

   ```bash
   ansible-playbook playbook.yml -i inventory.yml
   ```

   Дождитесь выполнения плейбука. Сервис и мониторинг будут доступны по своим соответствующим публичным IP-адресам балансировщика и сервера Zabbix.

### Настройка мониторинга

Чтобы настроить Zabbix Server:

1. Скопируйте файлы с шаблонами и списком хостов с помощью команды:

   ```bash
   BALANCER="<IP-адрес_балансировщика>"
   MONITORING="<IP-адрес_мониторинга>"
   KEY="ssh_keys/ansible_id_ed25519"

   scp -o ProxyCommand="ssh -W %h:%p -q -i $KEY -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null ansible@$BALANCER" -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -i $KEY ansible@$MONITORING:~/zbx_export_templates.yml ~/
   scp -o ProxyCommand="ssh -W %h:%p -q -i $KEY -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null ansible@$BALANCER" -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -i $KEY ansible@$MONITORING:~/zbx_export_hosts.yml ~/
   ```

1. По публичному IP-адресу мониторинга войдите в веб-интерфейс Zabbix Server с помощью учётных данных:

   * Логин: `Admin`.
   * Пароль: `14qrafzv`.

   > **Важно**
   > 
   > После подключения смените пароль.

1. На панели слева перейдите в раздел **Data collection** → **Templates**.
1. В верхнем правом углу нажмите **Import** и импортируйте загруженный ранее файл с шаблонами (`zbx_export_templates.yml`).
1. Перейдите в раздел **Hosts**.
1. В верхнем правом углу нажмите **Import** и импортируйте загруженный ранее файл со списком хостов (`zbx_export_hosts.yml`).

После этого настройка инфраструктуры будет завершена — сервис готов к работе.

## Резервное копирование и восстановление

### Скрипт

[`backup_copy.sh`](ansible/roles/database/files/backup_copy.sh):

```bash
#!/usr/bin/env bash

DB_DIR="$1"
ARCHIVE_NAME="$2"

# Проверка аргументов
if [ -z "$DB_DIR" ] || [ -z "$ARCHIVE_NAME" ]; then
  echo "Error: DB directory or archive name not provided."
  exit 1
fi

# Проверка существования директории БД
if [ ! -d "$DB_DIR" ]; then
  echo "Error: Database directory '$DB_DIR' does not exist."
  exit 1
fi

# Создаём папку для бэкапов, если её нет
ARCHIVE_DIR=$(dirname "$ARCHIVE_NAME")
mkdir -p "$ARCHIVE_DIR"

# Создаём архив
tar -czf "$ARCHIVE_NAME" -C "$(dirname "$DB_DIR")" "$(basename "$DB_DIR")"

# Проверка результата
if [ $? -eq 0 ]; then
  echo "Backup created: $ARCHIVE_NAME"
  exit 0
else
  echo "Error: Failed to create backup."
  exit 1
fi
```

Скрипт работает в связке с [service-файлом](./ansible/roles/database/files/backup_copy.service) и [таймером](./ansible/roles/database/files/backup_copy.timer).

Эта система автоматически создаёт безопасный бэкап всей СУБД PostgreSQL каждые 2 часа с помощью `pg_dumpall`.

* Таймер запускает задачу по расписанию (в `00:00`, `02:00` и т.д.).
* Сервис выполняет скрипт от имени пользователя `postgres`.
* Скрипт делает полный SQL-дамп всех баз и пользователей, сохраняет его с меткой времени и сжимает в `.gz`.

В результате в папке `/home/ansible/backups/` появляются читаемые, переносимые и безопасные бэкапы. Вся связка целиком работает на живой базе без риска повреждения.

### Восстановление

#### БД из бэкапа

Чтобы восстановить БД из бэкапа, выполните команду:

```bash
gunzip < <имя_файла_бэкапа>.sql.gz | sudo -u postgres psql
```

> **Внимание**
> 
> Восстановление перезапишет текущие объекты (роли, базы), будьте осторожны.

В случае если инцидент произошел только с файлами БД, но виртуальная машина работает стабильно, то вы можете ограничиться восстановлением файлов БД из бэкапа без пересоздания этой виртуальной машины.

#### Средства Yandex Cloud

Снимок диска в Yandex Cloud — это копия файловой системы диска на определенный момент времени. В облаке настроено создание снимков каждого диска по расписанию:

* Ежедневно в 00:00; хранятся последние 7 снимков.
* Еженедельно по субботам в 00:00; хранятся последние 10 снимков.
* Ежемесячно в первый день каждого месяца в 00:00; хранятся последние 10 снимков.

При необходимости вы можете [создать виртуальную машину](https://yandex.cloud/ru/docs/compute/operations/vm-create/create-from-snapshots) с дисками, восстановленными из имеющихся в каталоге снимков.

#### Использование Terraform и Ansible

Чтобы точечно пересоздать виртуальную машину, используйте команду `terraform taint`. Конфигурацию ресурсов вы можете найти в `~/wikimedia/terraform/main.tf`, а названия инстансов — в `~/wikimedia/terraform/variables.tf`. При пересоздании виртуальной машины отметьте для пересоздания и саму машину, и её диск. 

> **Пример**
> 
> 1. Чтобы пересоздать виртуальную машину `wikimedia-2`, используйте команду:
> 
>    ```bash
>    cd ~/wikimedia/terraform/
>    terraform taint 'resource.yandex_compute_disk.boot_disk["wikimedia-2"]'
>    terraform taint 'resource.yandex_compute_instance.vm["wikimedia-2"]'
>    . deploy_terraform.sh
>    ```
> 
> 1. Запустите плейбук Ansible с тегом `safe`, чтобы не затронуть все БД:
> 
>    ```
>    cd ../ansible/
>    ansible-playbook playbook.yml -i inventory.yml -t safe
>    ```
